**TDD approach: Write tests first (RED), then implement (GREEN).**

**Tests to write FIRST in `tests/test_slot_service.py`:**

1. `test_generate_slots_from_schedule` - Given schedule config (09:00-17:00, break 13:00-14:00, 15-min slots), generate_slots returns list of time strings: ["09:00", "09:15", ..., "12:45", "14:00", ..., "16:45"]. Should NOT include 13:00-13:45 (break period). Should NOT include 17:00 (end time is exclusive).

2. `test_generate_slots_excludes_break` - Verify no slot falls within break_start to break_end range.

3. `test_get_available_slots_excludes_booked` - Given 2 booked appointments for a date, get_available_slots returns all slots minus those 2.

4. `test_get_available_slots_excludes_locked` - Given a soft-locked slot (non-expired), it's excluded from available slots.

5. `test_get_available_slots_includes_expired_locks` - Given an expired soft-lock, that slot IS available (lock expired).

6. `test_same_day_excludes_past_slots` - For today's date, slots before current IST time are excluded.

7. `test_max_appointments_per_day` - When 50 appointments exist for a date, get_available_slots returns empty list.

8. `test_get_available_dates` - Returns list of dates (next 7 days) that have at least 1 available slot, excluding non-working days.

**Then implement `src/docbot/slot_service.py`:**

1. `def generate_slots(schedule: ScheduleConfig) -> list[str]` - Pure function. Generate all possible slot times from schedule config. Returns sorted list of "HH:MM" strings. Excludes break period. Slot at break_start is excluded; slot at break_end is included.

2. `async def get_available_slots(db, date_str: str, schedule: ScheduleConfig | None = None) -> list[str]` - Get available slots for a specific date.
   - Generate all slots from schedule config (use get_settings().schedule if not provided).
   - Query appointments table: SELECT slot_time FROM appointments WHERE appointment_date = ? AND status NOT IN ('CANCELLED', 'REFUNDED')
   - Query slot_locks table: SELECT slot_time FROM slot_locks WHERE appointment_date = ? AND locked_until > utc_now()
   - Remove booked and locked slots from generated slots.
   - If date is today (IST), remove slots where IST time has already passed.
   - If total non-cancelled appointments for date >= max_appointments_per_day, return [].
   - Return remaining available slots as sorted list.

3. `async def get_available_dates(db, days_ahead: int = 7, schedule: ScheduleConfig | None = None) -> list[str]` - Get dates with available slots.
   - Start from today (IST). Include today if working day with available future slots.
   - Check each date for next `days_ahead` days.
   - Only include dates where: (a) day-of-week is in working_days, (b) at least 1 slot available.
   - Return list of "YYYY-MM-DD" strings.

**TDD approach: Write tests first (RED), then implement (GREEN).**

**Tests to write FIRST in `tests/test_booking_service.py`:**

1. `test_lock_slot_success` - lock_slot for an available slot creates entry in slot_locks table with 10-min expiry. Returns True.

2. `test_lock_slot_already_locked` - lock_slot for a slot that's already locked by another phone returns False (PRIMARY KEY constraint).

3. `test_lock_slot_expired_lock_allows_relock` - If an expired lock exists, lock_slot cleans it up and creates new lock. Returns True.

4. `test_release_lock` - release_lock removes the slot_lock entry for a phone+date+time.

5. `test_create_appointment_offline` - create_appointment with consultation_type="offline" creates appointment with status "CONFIRMED" (no payment needed). Returns appointment dict.

6. `test_create_appointment_online` - create_appointment with consultation_type="online" creates appointment with status "PENDING_PAYMENT". Returns appointment dict.

7. `test_create_appointment_releases_lock` - After creating appointment, the soft-lock for that slot is released.

8. `test_create_appointment_duplicate_slot_fails` - Creating appointment for an already-booked slot (existing non-cancelled appointment) raises ValueError.

9. `test_cleanup_expired_locks` - cleanup_expired_locks removes locks past their locked_until time. Returns count removed.

**Then implement `src/docbot/booking_service.py`:**

1. `async def lock_slot(db, phone: str, date_str: str, slot_time: str, ttl_minutes: int = 10) -> bool`
   - First cleanup_expired_locks for this date+slot (remove if expired).
   - Try INSERT into slot_locks with locked_until = utc_now() + ttl_minutes.
   - If INSERT succeeds, commit and return True.
   - If IntegrityError (PRIMARY KEY violation), return False (slot already locked).

2. `async def release_lock(db, phone: str, date_str: str, slot_time: str) -> None`
   - DELETE from slot_locks WHERE appointment_date=? AND slot_time=? AND locked_by_phone=?.
   - Commit.

3. `async def create_appointment(db, phone: str, name: str, age: int, gender: str, consultation_type: str, date_str: str, slot_time: str, language: str = "en") -> dict`
   - Verify slot is not already booked (SELECT from appointments WHERE date+slot AND status NOT IN CANCELLED/REFUNDED). Raise ValueError if taken.
   - Determine initial status: "CONFIRMED" for offline, "PENDING_PAYMENT" for online. Use state_machine.AppointmentStatus values.
   - Generate UUID for appointment id.
   - INSERT into appointments table with all fields.
   - Release the soft-lock for this slot (it's now a real booking).
   - Commit and return the appointment as dict.

4. `async def cleanup_expired_locks(db, date_str: str | None = None, slot_time: str | None = None) -> int`
   - DELETE from slot_locks WHERE locked_until < utc_now(). Optionally filter by date and slot.
   - Return number of rows deleted.

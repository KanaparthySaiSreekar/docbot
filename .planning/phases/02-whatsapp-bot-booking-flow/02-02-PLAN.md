**Schema migration `db/002_patients_conversations.sql`:**

```sql
-- Patients table for language preference persistence (BOT-03)
CREATE TABLE IF NOT EXISTS patients (
    phone TEXT PRIMARY KEY,           -- E.164 format e.g. "919876543210"
    language TEXT NOT NULL DEFAULT 'en',  -- 'en', 'te', 'hi'
    name TEXT,                        -- Populated during booking
    created_at TEXT NOT NULL,         -- UTC ISO 8601
    updated_at TEXT NOT NULL          -- UTC ISO 8601
);

-- Conversations table for tracking booking flow state (BOT-09)
CREATE TABLE IF NOT EXISTS conversations (
    phone TEXT PRIMARY KEY,           -- One active conversation per phone (BOT-09)
    state TEXT NOT NULL,              -- Current step in flow
    data TEXT,                        -- JSON blob for accumulated booking data
    started_at TEXT NOT NULL,         -- UTC ISO 8601
    updated_at TEXT NOT NULL,         -- UTC ISO 8601
    expires_at TEXT NOT NULL          -- UTC ISO 8601 - conversation timeout
);

CREATE INDEX IF NOT EXISTS idx_conversations_expires ON conversations(expires_at);
```

**Patient store `src/docbot/patient_store.py`:**
Functions (all async, take `db: aiosqlite.Connection`):

1. `async def get_or_create_patient(db, phone: str) -> dict` - Get patient by phone. If not exists, create with default language 'en'. Return dict with phone, language, name, created_at, updated_at.

2. `async def get_language(db, phone: str) -> str` - Get language for phone. Returns 'en' if patient doesn't exist.

3. `async def set_language(db, phone: str, language: str) -> None` - Update language preference. Validates language is one of ('en', 'te', 'hi'). Creates patient if not exists.

4. `async def update_patient_name(db, phone: str, name: str) -> None` - Update patient name (populated during booking flow).

**Conversation state `src/docbot/conversation.py`:**

Define conversation states as string constants (not Enum, for JSON serialization simplicity):
```python
# Conversation states for the booking flow

MAIN_MENU = "MAIN_MENU"               # At main menu

ENTER_GENDER = "ENTER_GENDER"         # Choosing gender
CONFIRM_BOOKING = "CONFIRM_BOOKING"   # Confirming booking details
```

Functions (all async, take `db: aiosqlite.Connection`):

1. `async def get_conversation(db, phone: str) -> dict | None` - Get active conversation for phone. Returns None if no active conversation or if expired. If expired, delete it automatically.

2. `async def start_conversation(db, phone: str, state: str = LANGUAGE_SELECT) -> dict` - Start new conversation. If one already exists for this phone (BOT-09), delete it first. Set expires_at to 30 minutes from now. Data starts as empty JSON `{}`. Return conversation dict.

3. `async def update_conversation(db, phone: str, state: str, data: dict | None = None) -> dict` - Update conversation state and optionally merge new data into existing data blob. Reset expires_at to 30 minutes from now (rolling expiry). Return updated conversation.

4. `async def end_conversation(db, phone: str) -> None` - Delete conversation for phone (booking complete or cancelled).

5. `async def cleanup_expired_conversations(db) -> int` - Delete all expired conversations. Return count deleted. (Called periodically or on access.)

- Conversation data blob stores accumulated booking info as JSON: `{"consultation_type": "online", "appointment_date": "2026-02-10", "slot_time": "10:00", "patient_name": "...", "patient_age": 25, "patient_gender": "Male"}`
- Use `json.dumps`/`json.loads` for data serialization.
- Use `timezone_utils.utc_now()` for all timestamps.

  <action>
Create `src/docbot/bot_handler.py` - the central message handler that routes incoming messages through the conversation flow.

**Main entry point:**

```python
async def handle_message(parsed_message: dict) -> None:
```

Takes the parsed message dict from webhook.py: `{"from": phone, "type": msg_type, "message_id": wamid, "button_id": id_or_none, "button_title": title_or_none, "text": text_or_none, "timestamp": ts}`

**Flow logic:**

1. Get patient language: `patient_store.get_language(db, phone)`
2. Get active conversation: `conversation.get_conversation(db, phone)`

3. **If no conversation exists (first message or expired):**
   - Check if patient exists. If new patient (no language set yet), start LANGUAGE_SELECT flow.
   - If returning patient (has language), start MAIN_MENU flow.
   - If conversation was expired, send `session_expired` message first, then restart.

4. **Route by conversation state:**

   **LANGUAGE_SELECT:**
   - Send welcome message with 3 language buttons: English, Telugu, Hindi.
   - On button reply: save language via patient_store.set_language(), send language_set confirmation, advance to MAIN_MENU.
   - On invalid input: resend language buttons with invalid_input message.

   **MAIN_MENU:**
   - Send main menu with 3 buttons: Book Appointment, View Appointment, Cancel Appointment.
   - Also include Contact Clinic option. Since max 3 buttons, use: Book Appointment, Cancel Appointment, Contact Clinic. View Appointment will be part of a "More" flow or combined with Cancel later. For Phase 2, implement 3 buttons: "Book Appointment", "Cancel Appointment" (placeholder - sends "Coming soon" message), "Contact Clinic".
   - On "Book Appointment": advance to SELECT_TYPE.
   - On "Contact Clinic": send clinic contact info from config, stay at MAIN_MENU.
   - On "Cancel Appointment": send "Coming soon in next update" message, stay at MAIN_MENU.
   - On invalid input: resend menu with invalid_input message.

   **SELECT_TYPE:**
   - Send consultation type selection with 2 buttons: Online, Offline.
   - On button reply: store consultation_type in conversation data, advance to SELECT_DATE.
   - On invalid input: resend type buttons.

   **SELECT_DATE:**
   - Get available dates via slot_service.get_available_dates().
   - If no dates: send no_dates_available message, end conversation, return to MAIN_MENU.
   - Send dates as interactive LIST message (dates can exceed 3, so use list not buttons).
   - Each row: id = date string "YYYY-MM-DD", title = formatted date "Wed, 12 Feb", description = "X slots available".
   - On list reply: store appointment_date in conversation data, lock step not needed yet. Advance to SELECT_SLOT.
   - On invalid input: resend date list.

   **SELECT_SLOT:**
   - Get available slots via slot_service.get_available_slots() for the selected date.
   - If no slots: send no_slots_available message, go back to SELECT_DATE.
   - Send slots as interactive LIST message (slots will be >3).
   - Each row: id = "HH:MM", title = "HH:MM AM/PM" (formatted for display), description = "Available".
   - On list reply: attempt to lock slot via booking_service.lock_slot(). If lock fails (slot taken), send slot_taken message and resend slot list. If lock succeeds, store slot_time in conversation data, advance to ENTER_NAME.
   - On invalid input: resend slot list.

   **ENTER_NAME:**
   - Send enter_name prompt (text message, not buttons - this is the one step requiring text input).
   - On text reply: validate non-empty, store patient_name in conversation data, advance to ENTER_AGE.
   - On empty/invalid: resend prompt.

   **ENTER_AGE:**
   - Send enter_age prompt (text message).
   - On text reply: validate is numeric and between 0-150. Store patient_age in conversation data, advance to ENTER_GENDER.
   - On invalid: resend prompt with error message.

   **ENTER_GENDER:**
   - Send gender selection with 3 buttons: Male, Female, Other.
   - On button reply: store patient_gender in conversation data, advance to CONFIRM_BOOKING.
   - On invalid: resend buttons.

   **CONFIRM_BOOKING:**
   - Send booking summary with all details formatted: type, date, time, name, age, gender.
   - Send 2 buttons: Confirm, Cancel.
   - On "Confirm":
     - Re-verify slot availability (TIME-03: recalculate before final confirmation).
     - If slot still available: call booking_service.create_appointment(). Send booking_confirmed message. End conversation.
     - If slot no longer available: send slot_taken message, release lock, go back to SELECT_SLOT.
   - On "Cancel": release lock, send booking_cancelled_by_user message. End conversation.
   - On invalid: resend confirmation.

**Important implementation details:**
- All message sending uses whatsapp_client functions (send_text, send_buttons, send_list).
- All messages use i18n.get_message() with patient's language.
- Use logging throughout for debugging conversation flow.
- Wrap entire handler in try/except: on any unhandled error, send error_generic message and end conversation. Log error with full traceback.
- Use `async for db in get_db()` pattern for database access within the handler.
- Format slot times for display: convert "09:00" to "9:00 AM", "14:30" to "2:30 PM".

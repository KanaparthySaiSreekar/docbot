<task type="auto">
  <name>Task 1: Create dashboard API module with appointment endpoints</name>
  <files>src/docbot/dashboard_api.py</files>
  <action>
Create new FastAPI router module `dashboard_api.py` with these endpoints:

1. **GET /api/appointments**
   - Query params: `date_from` (optional), `date_to` (optional), `status` (optional)
   - Default: today's date to 7 days ahead
   - Returns JSON array of appointments with all fields from appointments table
   - Join with refunds table to include refund_status where applicable
   - Include google_meet_link for online appointments
   - Order by appointment_date ASC, slot_time ASC

2. **GET /api/appointments/history**
   - Returns past appointments (appointment_date < today)
   - Query params: `limit` (default 50), `offset` (default 0)
   - Order by appointment_date DESC, slot_time DESC

3. **GET /api/refunds/failed**
   - Returns refunds with status = 'PENDING' or 'FAILED'
   - Include appointment details (patient name, date, amount)
   - Order by created_at DESC

4. **GET /api/settings**
   - Returns current ScheduleConfig from settings
   - Format: { working_days, start_time, end_time, break_start, break_end, slot_duration_minutes }

All endpoints must:
- Use `require_auth` dependency from auth.py (raises 401 if unauthenticated)
- Use `get_db` dependency for database access
- Return proper JSON responses with Pydantic models
- Log access without PII (log appointment counts, not patient names)

Response models (define in dashboard_api.py):
- AppointmentListItem: id, patient_name, patient_age, patient_gender, patient_phone (masked), consultation_type, appointment_date, slot_time, status, google_meet_link, refund_status
- RefundListItem: id, appointment_id, patient_name, appointment_date, amount_paise, status, retry_count, created_at
- SettingsResponse: working_days, start_time, end_time, break_start, break_end, slot_duration_minutes

Phone masking: Show last 4 digits only (e.g., "****1234")

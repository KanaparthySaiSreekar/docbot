<task type="auto">
  <name>Task 1: Google OAuth flow with Authlib</name>
  <files>src/docbot/auth.py</files>
  <action>
    1. Create src/docbot/auth.py with a FastAPI router (prefix="/auth"):

    2. Configure Authlib OAuth client for Google:
       - Use authlib.integrations.starlette_client.OAuth
       - Register google provider with client_id and client_secret from Settings
       - Server metadata URL: https://accounts.google.com/.well-known/openid-configuration
       - Scopes: openid, email, profile

    3. Session middleware setup:
       - Use Starlette's SessionMiddleware with Settings.auth.session_secret_key
       - Set session cookie: httponly=True, samesite="lax", secure=True (in prod)
       - Session max_age: None (never expires, per CONTEXT decision)
       - AUTH-02: Session persists across browser sessions via cookie

    4. Routes:
       - GET /auth/login: Redirects to Google OAuth authorization URL
       - GET /auth/callback: Handles OAuth callback, extracts user info (email, name, picture), stores in session, redirects to /dashboard (or / for now)
       - GET /auth/logout: Clears session, redirects to landing page
       - If OAuth fails at callback: redirect to /auth/error with error message

    5. Auth dependency (require_auth):
       - FastAPI dependency that checks session for user info
       - If no session: redirect to landing page (GET requests) or return 401 (API requests)
       - AUTH-03: No allowlist check -- any Google account works (rely on Cloudflare tunnel obscurity per CONTEXT)
       - Returns user dict with email, name, picture

    6. Error handling:
       - Catch all OAuth exceptions (network errors, invalid tokens, etc.)
       - Log the error at ERROR level with details
       - Redirect to error page, never show raw error to user
  </action>
  <verify>
    Import check: `uv run python -c "from docbot.auth import router, require_auth; print('Auth module OK')"` should print without errors.
  </verify>
  <done>OAuth flow implemented: /auth/login redirects to Google, /auth/callback handles response and creates session, require_auth dependency protects routes.</done>
</task>

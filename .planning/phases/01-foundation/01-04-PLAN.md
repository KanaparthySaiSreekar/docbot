    1. Create scripts/admin_override.py -- a CLI tool for direct database corrections (OPS-04):
       - Uses argparse for subcommands
       - Subcommands:
         a. `list-appointments --date YYYY-MM-DD` -- list all appointments for a date
         b. `get-appointment --id UUID` -- show full appointment details
         c. `update-status --id UUID --status STATUS` -- change appointment status (validates against state machine)
         d. `release-locks --expired` -- release all expired slot locks
         e. `run-sql --query "SELECT ..."` -- execute arbitrary read-only SQL (SELECT only)
       - Connects directly to SQLite DB file (path from command line arg --db)
       - Imports state_machine.transition() for status validation
       - Logs all modifications to stdout with timestamps
       - Requires --confirm flag for any write operations (safety)
       - Print results as formatted tables

    2. Create scripts/backup_verify.sh -- backup and verification script (OPS-05):
       - Takes DB path as argument (default: data/docbot.db)
       - Creates timestamped backup: data/backups/docbot_YYYYMMDD_HHMMSS.db
       - Runs SQLite integrity check on backup: `sqlite3 backup.db "PRAGMA integrity_check;"`
       - Verifies row counts match between source and backup for each table
       - Prints verification summary (table name, source rows, backup rows, match status)
       - Exits with non-zero if any check fails
       - OPS-05: This verifies backup integrity, not just that backup was created
       - Cleanup: delete backups older than 7 days

       - Future phases will add actual alerting (email, webhook, etc.)
